# Search and Navigation UX

**Section:** 10 Product

[Previous: List Views](./10-list-views.md) | [Index](./01-index.md) | [Next: History, Record Pages, and Chatter](./12-history-record-pages-and-chatter.md)

This note covers the user-facing navigation and search model.

## Global Search

Global search works across objects the user can read.

Ranking priority is:

1. exact
2. starts with
3. contains

## Search Experience

The product supports:

- header search entry
- command-style quick search feel
- autocomplete-style results
- full results page with pagination

## Search Filter

Users can narrow the search page by object.

## Search Exclusions

TextArea and File fields are excluded from search.

## Navigation

Navigation is app-based in the standard area.

Users typically move through:

- app switcher
- app dashboard
- app sidebar objects
- list views
- record detail pages

## Record Detail UX

Record detail pages commonly expose:

- details
- related data
- history
- chatter when enabled
- inline file preview when relevant
